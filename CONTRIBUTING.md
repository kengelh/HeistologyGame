# Contributing to Heistology Beta

First off, thank you for considering contributing to Heistology Beta! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs

-   **Check existing issues:** See if the bug has already been reported.
-   **Include details:** Provide a clear description, steps to reproduce, and any relevant logs or screenshots.

### Suggesting Enhancements

-   **Be specific:** Describe the feature you'd like to see and why it would be beneficial.
-   **Check for duplicates:** Ensure the enhancement hasn't been suggested before.

### Submitting Pull Requests

1.  **Fork the repository.**
2.  **Create a new branch:** `git checkout -b feature/your-feature-name` or `git checkout -b fix/your-fix-name`.
3.  **Make your changes.**
4.  **Write tests:** If applicable, add or update tests to cover your changes.
5.  **Commit with clear messages:** Explain what you did and why.
6.  **Push to your fork:** `git push origin your-branch-name`.
7.  **Submit a Pull Request.**

## Coding Standards

-   **TypeScript:** Always use TypeScript and ensure your code is type-safe.
-   **Tailwind CSS:** Use Tailwind for styling whenever possible.
-   **React Best Practices:** Follow hook rules, use functional components, and keep components modular.
-   **Comments:** Write clear comments for complex logic, but aim for self-documenting code.
